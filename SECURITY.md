# Security Policy

Found a security vulnerability in Bede's code? Please use GitHub's private
vulnerability reporting for this repository (repo → **Security** tab →
**Report a vulnerability**) rather than a public issue, so it reaches
maintainers before any details are public.

For everything else security-related — the full incident response
procedure, how to read your own instance's audit log, breach-notification
guidance, and who's accountable for what — see
[`docs/INCIDENT_RESPONSE.md`](docs/INCIDENT_RESPONSE.md). For Bede's
overall security architecture and AIUC-1/SOC 2 compliance posture, see
[`docs/SECURITY.md`](docs/SECURITY.md).
